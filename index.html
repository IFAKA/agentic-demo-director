What happens

The agent edits recipes. The recorder executes them.

Typed scenarios

Reusable `.demo.ts` files describe routes, viewport, timing, assertions, and final video format.

Visible gestures

Tap ripples and drag trails are injected by the recorder, so every app gets the same polished touch language.

Deterministic browser

Each run uses a fresh Playwright context with fixed viewport, locale, timezone, color scheme, and storage.

Validated exports

ffprobe checks dimensions, frame rate, duration, codec, and audio before the run is considered complete.

Scenario

A minimal walkthrough recipe

import { defineDemo } from "@faka/demo-director";

export default defineDemo({
  name: "main",
  startUrl: "/demo/main?demo=1",
  output: "dist/demo/main.mp4",
  server: {
    command: "npm run dev",
    url: "http://127.0.0.1:3000",
  },
  format: {
    width: 1080,
    height: 1350,
    fps: 30,
    duration: { min: 20, max: 40 },
    crop: "social-4x5",
    cursor: "hidden",
  },
  steps: async ({ page, gesture, expect }) => {
    await expect(page.getByRole("heading", { name: "Home" })).toBeVisible();
    await gesture.tap(page.getByRole("button", { name: "Start" }));
    await expect(page.getByText("Success")).toBeVisible();
    await gesture.wait(1500);
  },
});
